# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in the WQM-1 firmware — for example, issues with LoRa key handling, AES-128 implementation, or data pipeline security — please report it responsibly.

**Do not open a public GitHub issue.**

Instead, email **security@bluesignal.xyz** with:

- A description of the vulnerability
- Steps to reproduce
- Affected firmware version(s)
- Any suggested fix (optional but appreciated)

We will acknowledge your report within 48 hours and work with you on a fix before any public disclosure.

## Scope

This policy covers the open-source firmware and hardware designs in this repository. For vulnerabilities in the BlueSignal cloud platform or WaterQuality.Trading marketplace, contact security@bluesignal.xyz as well, but note those systems are maintained separately.
